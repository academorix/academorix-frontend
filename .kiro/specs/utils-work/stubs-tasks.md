# Stubs Tasks

Author 69 template stubs across 6 language/category folders, plus wire every
logical name into `StubRegistry.php`.

**File-tree ownership**:

- `tools/cli/src/Stubs/stubs/**` (69 stub files)
- `tools/cli/src/Stubs/StubRegistry.php` (registry file — CLI structure lives in
  `cli-tasks.md` but this file's `defaultStubs()` map is owned here so the two
  lands cleanly)

**Depends on**: `cli-tasks.md` (StubRegistry.php exists; this task writes its
`defaultStubs()` map + creates the stub files it references).

**No git operations.**

---

## Global token convention

Every stub uses `{{ tokenName }}` (camelCase inside double braces with a space
either side). Reserved tokens across all stub categories:

- `{{ moduleName }}` — kebab-case module identifier
- `{{ tier }}` — Tier 1/2/3 name
- `{{ pascalName }}` / `{{ camelName }}` / `{{ kebabName }}` — case-cast
  variants
- `{{ namespace }}` — PHP namespace (or JS import root)
- `{{ description }}` — one-liner description
- `{{ longDescription }}` — multi-paragraph description
- `{{ date }}` — ISO date at generation time
- `{{ version }}` — Stackra generator version string

Every stub ends with an HTML/PHP-comment footer:
`Generated by 'stackra make:<command>' <version>` so downstream tooling can
trace back to the emitter.

## Recommended parallelization

The 69 stubs split into 5 disjoint sub-agent lanes:

- **Lane A — PHP (26 stubs)** — laravel-feature-builder
- **Lane B — React (18 stubs)** — heroui-ui-builder
- **Lane C — Native (8 stubs)** — heroui-native-builder
- **Lane D — TypeScript + Config (10 stubs)** — framework-core-builder
- **Lane E — Docs (7 stubs)** — docs-adr-steward

All 5 lanes write to disjoint sub-folders under `stubs/`. The `StubRegistry.php`
edit happens ONCE at the end after all lanes report back, to avoid the classic
"5 agents editing the same file" merge disaster.
