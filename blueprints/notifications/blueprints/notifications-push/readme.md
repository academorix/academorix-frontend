# notifications-push

Push notification channel. Wave 1. Owns the `PushSubscription` entity + own
`MultipleInstanceManager` for FCM / APNs / Expo / OneSignal.

## 1. What this module owns

| Concern                             | Owned artefact                                                                |
| ----------------------------------- | ----------------------------------------------------------------------------- |
| Device tokens per user per platform | `PushSubscription`                                                            |
| Provider abstraction                | `PushTransportManager` (extends `Illuminate\Support\MultipleInstanceManager`) |
| Token validation before persist     | `TokenValidator` binding (dry-run to provider)                                |
| Invalid-token cleanup               | `PruneExpiredSubscriptionsJob` + `PushInvalidToken` event                     |

## 2. Providers

- **FCM** (Firebase Cloud Messaging) — Android + Web push. Uses HTTP v1 API with
  OAuth 2.0 service account. Free.
- **APNs** (Apple Push Notification service) — iOS + macOS + iPadOS. Token-based
  auth (p8 key) preferred over cert-based. Free.
- **Expo** — React Native. Expo push tokens routed through Expo servers to
  FCM/APNs. Free tier + paid volume.
- **OneSignal** — Enterprise. Cross-platform + advanced segmentation + rich
  notifications. Metered.

## 3. Delivery flow

```
NotificationDispatched (channel=push)
       │
       ▼
  HandlePushDispatch listener
       │
       ▼
  SendPushJob (queue=notifications)
       │
       ├── Query PushSubscription rows: where user_id + is_active + tenant_id + application_id
       ├── For each subscription:
       │     - Resolve provider driver (fcm/apns/expo/onesignal)
       │     - Build payload: alert + badge + sound + custom data
       │     - Provider.send(token, payload)
       │     - On invalid_token response: fire PushInvalidToken → mark subscription expired
       │     - On success: capture provider_message_id, create NotificationDelivery
       ▼
  Fire PushSent (delivery.state=sent, provider_message_id captured)
       │
       │  Provider feedback (later, via webhook)
       ▼
  webhook::InboundWebhookReceived (namespace=notifications-push, provider=fcm|apns)
       │
       ▼
  IngestPushProviderWebhookListener
       │
       ▼
  PushDelivered / PushInvalidToken / PushOpened
```

## 4. Device token security

`PushSubscription.device_token` is **RESTRICTED** tier — encrypted at rest via
AWS KMS. Never returned by API responses (hidden array + explicit exclusion in
policy responses). Rotated by client on app launch — expired tokens auto-marked
by `PushInvalidToken` handler.

## 5. COPPA VPC gate

Minor recipients (`recipient.consent_age_gate = minor_under_consent_age`) have
push notifications hard-suppressed on marketing-priority categories until
parental consent captured (via compliance module). Transactional categories
still deliver.

## 6. Files

Standard blueprint. Schemas: 1 entity (`push-subscription`).

## 7. What this module does NOT do

- **Doesn't own provider API keys.** Doppler-backed via
  `config/notifications-push.php`.
- **Doesn't own the notification content.** Templates come from `notifications`
  core; push renders subject/body as `alert.title` + `alert.body`.
- **Doesn't own the click-through tracking.** Click URLs generated by
  notifications core; push includes them in `custom_data.click_url`.
- **Doesn't own inbound webhook receiving.** Universal `webhook` module owns
  `POST /webhooks/inbound/{namespace}/{provider}`.
